# opnsense_system_tunable (Resource)

Manages a persistent system tunable on OPNsense.

Tunables can affect kernel, network, firewall, storage, or service behavior. Test tunable changes on a disposable appliance before applying them to production. The provider applies changes through the OPNsense tunables reconfigure endpoint, but some tunables may still require a reboot or subsystem reload before every runtime effect is visible.

The provider manages only persistent tunable configuration fields: `tunable`, `value`, and `description`. OPNsense readback metadata such as default value and tunable type is not managed as Terraform desired state.

## Example Usage

```terraform
resource "opnsense_system_tunable" "msgbuf_timestamp" {
  tunable     = "kern.msgbuf_show_timestamp"
  value       = "1"
  description = "Show timestamps in the kernel message buffer"
}
```
