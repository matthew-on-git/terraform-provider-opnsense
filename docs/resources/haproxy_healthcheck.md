# opnsense_haproxy_healthcheck (Resource)

Manages an HAProxy health check on OPNsense. Health checks verify server availability in backend pools. Supports HTTP, TCP, SSL, and other check types. Requires the `os-haproxy` plugin.

## Example Usage

```terraform
# HTTP health check on a custom path
resource "opnsense_haproxy_healthcheck" "http_check" {
  name        = "http-health"
  type        = "http"
  http_method = "get"
  http_uri    = "/health"
  interval    = "5s"
}

# Simple TCP health check
resource "opnsense_haproxy_healthcheck" "tcp_check" {
  name     = "tcp-health"
  type     = "tcp"
  interval = "3s"
}
```

## Schema

### Required

- `name` (String) Name of the health check.

### Optional

- `check_port` (String) Port to use for health checks. Empty to use the server port.
- `description` (String) Description of the health check.
- `force_ssl` (Boolean) Force SSL for health checks. Defaults to `false`.
- `http_method` (String) HTTP method for health check: `options`, `head`, `get`, `put`, `post`, `delete`, `trace`.
- `http_uri` (String) URI path for HTTP health check. Defaults to `/`.
- `http_version` (String) HTTP version: `http10`, `http11`, `http2`.
- `interval` (String) Check interval (e.g., `2s`, `500ms`, `1m`).
- `type` (String) Check type: `tcp`, `http`, `agent`, `ldap`, `mysql`, `pgsql`, `redis`, `smtp`, `esmtp`, `ssl`.

### Read-Only

- `id` (String) UUID of the HAProxy health check in OPNsense.
