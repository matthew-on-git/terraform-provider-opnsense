# opnsense_ddclient_settings (Resource)

Manages Dynamic DNS daemon settings on OPNsense. This is a singleton resource for `/api/dyndns/settings`; `terraform destroy` only removes the singleton from Terraform state and does not reset appliance settings. Requires `os-ddclient`.

## Example Usage

```terraform
resource "opnsense_ddclient_settings" "daemon" {
  enabled    = true
  backend    = "opnsense"
  interval   = 300
  verbose    = false
  allow_ipv6 = true
}

resource "opnsense_ddclient_account" "cloudflare" {
  enabled   = true
  service   = "cloudflare"
  hostnames = "www.example.com"
  username  = "cloudflare-account@example.com"
  password  = var.cloudflare_api_token

  depends_on = [opnsense_ddclient_settings.daemon]
}
```

## Schema

### Optional

- `allow_ipv6` (Boolean) Allow IPv6 address updates. Defaults to `false`.
- `backend` (String) Dynamic DNS backend: `ddclient` or native OPNsense backend (`opnsense`). Defaults to `opnsense`.
- `enabled` (Boolean) Enable the Dynamic DNS daemon. Defaults to `true`.
- `interval` (Number) Daemon update interval in seconds (`daemon_delay`). Must be between `1` and `86400`. Defaults to `300`.
- `verbose` (Boolean) Enable verbose Dynamic DNS daemon logging. Defaults to `false`.

### Read-Only

- `id` (String) Synthetic identifier for this singleton (always `ddclient-settings`).
