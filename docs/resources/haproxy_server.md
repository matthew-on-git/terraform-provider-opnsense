# opnsense_haproxy_server (Resource)

Manages an HAProxy backend server on OPNsense. Servers define the upstream targets that HAProxy routes traffic to. Requires the `os-haproxy` plugin to be installed on the OPNsense appliance.

## Example Usage

```terraform
# Manage a basic HAProxy backend server
resource "opnsense_haproxy_server" "web1" {
  name    = "web-server-1"
  address = "10.0.0.10"
  port    = 80
  weight  = 100
}

# Manage an SSL-enabled backend server
resource "opnsense_haproxy_server" "api1" {
  name       = "api-server-1"
  address    = "10.0.0.20"
  port       = 443
  ssl        = true
  ssl_verify = true
  weight     = 50
}

# Manage a backup server (failover only)
resource "opnsense_haproxy_server" "backup" {
  name    = "backup-server"
  address = "10.0.0.99"
  port    = 80
  mode    = "backup"
}
```

## Schema

### Required

- `address` (String) IP address or hostname of the backend server.
- `name` (String) Name of the server.
- `port` (Number) Port number of the backend server (1-65535).

### Optional

- `description` (String) Description of the server.
- `enabled` (Boolean) Whether this server is enabled. Defaults to `true`.
- `mode` (String) Server mode: `active`, `backup`, or `disabled`.
- `ssl` (Boolean) Whether to use SSL/TLS for connections to this server. Defaults to `false`.
- `ssl_verify` (Boolean) Whether to verify the server's SSL certificate. Defaults to `true`.
- `weight` (Number) Load balancing weight (0-256). Higher values receive more traffic.

### Read-Only

- `id` (String) UUID of the HAProxy server in OPNsense.

## Import

Import is supported using the server UUID from OPNsense.

```shell
# Import an existing HAProxy server by its UUID
```
