# opnsense_interface_lagg (Resource)

Manages a LAGG interface through the OPNsense `interfaces/lagg_settings` API.

Use this resource only with free, non-management physical interfaces. Do not use
the active WAN or LAN management interface as a LAGG member in acceptance tests
or production changes unless you have an out-of-band recovery path.

The OPNsense API auto-assigns the LAGG device name, such as `lagg0`. The
provider reads that value back into `device` after creation.

## Example Usage

```terraform
resource "opnsense_interface_lagg" "uplink" {
  members     = ["em2", "em3"]
  protocol    = "failover"
  description = "Terraform managed LAGG"
}
```
