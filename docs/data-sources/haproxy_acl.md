# opnsense_haproxy_acl (Data Source)

Look up an existing HAProxy ACL on OPNsense. Use this data source to read attributes by UUID without importing the object into Terraform management.

## Example Usage

```terraform
# Look up an existing haproxy acl by UUID
data "opnsense_haproxy_acl" "example" {
  id = "12345678-1234-1234-1234-123456789012"
}

output "haproxy_acl_id" {
  value = data.opnsense_haproxy_acl.example.id
}
```

### Read-Only

- `custom_acl` (String) Raw HAProxy condition (pass-through).
- `description` (String) Description of the ACL.
- `expression` (String) Match expression type (e.g., 'hdr_beg', 'hdr', 'path_beg', 'path', 'ssl_fc_sni', 'ssl_sni', 'src', 'nbsrv', 'custom_acl').
- `hdr` (String) HTTP Host header exact match.
- `hdr_beg` (String) HTTP Host header starts with this value.
- `hdr_end` (String) HTTP Host header ends with this value.
- `name` (String) Name of the ACL rule.
- `nbsrv_backend` (String) UUID of the backend to check for minimum usable servers.
- `negate` (Boolean) Invert the match condition. Defaults to 'false'.
- `path` (String) URL path exact match.
- `path_beg` (String) URL path starts with this value.
- `src` (String) Source IP address match.
- `ssl_fc_sni` (String) SNI TLS extension matches (locally deciphered).
- `ssl_sni` (String) SNI TLS extension matches (TCP inspection).
