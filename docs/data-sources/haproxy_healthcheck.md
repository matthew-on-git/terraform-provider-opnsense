# opnsense_haproxy_healthcheck (Data Source)

Look up an existing HAProxy health check on OPNsense. Use this data source to read attributes by UUID without importing the object into Terraform management.

## Example Usage

```terraform
# Look up an existing haproxy healthcheck by UUID
data "opnsense_haproxy_healthcheck" "example" {
  id = "12345678-1234-1234-1234-123456789012"
}

output "haproxy_healthcheck_id" {
  value = data.opnsense_haproxy_healthcheck.example.id
}
```

### Required

- `id` (String) UUID to look up.

### Read-Only

- `check_port` (String) Port to use for health checks. Empty to use the server port.
- `description` (String) Description of the health check.
- `force_ssl` (Boolean) Force SSL for health checks. Defaults to 'false'.
- `http_method` (String) HTTP method for health check: 'options', 'head', 'get', 'put', 'post', 'delete', 'trace'.

- `http_version` (String) HTTP version: 'http10', 'http11', 'http2'.
- `interval` (String) Check interval (e.g., '2s', '500ms', '1m').
- `name` (String) Name of the health check.
- `type` (String) Check type: 'tcp', 'http', 'agent', 'ldap', 'mysql', 'pgsql', 'redis', 'smtp', 'esmtp', 'ssl'.
