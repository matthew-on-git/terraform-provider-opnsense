# opnsense_ipsec_remote (Data Source)

Look up an existing IPsec remote identity on OPNsense. Use this data source to read attributes by UUID without importing the object into Terraform management.

## Example Usage

```terraform
# Look up an existing ipsec remote by UUID
data "opnsense_ipsec_remote" "example" {
  id = "12345678-1234-1234-1234-123456789012"
}

output "ipsec_remote_id" {
  value = data.opnsense_ipsec_remote.example.id
}
```

## Schema

### Required

- `id` (String) UUID to look up.

### Read-Only

- `auth` (String) Authentication method: 'psk', 'pubkey', 'eap-tls', 'eap-mschapv2', 'xauth-pam', or 'eap-radius'. Defaults to 'psk'.
- `cacerts` (Set of String) UUIDs of CA certificates used to validate the peer.
- `certs` (Set of String) UUIDs of certificates accepted for authentication.
- `connection_id` (String) UUID of the IPsec connection this entry belongs to.
- `description` (String) Description of the remote authentication entry.
- `eap_id` (String) EAP identity, when an EAP authentication method is used.
- `enabled` (Boolean) Whether this remote authentication entry is enabled. Defaults to 'true'.
- `groups` (Set of String) Local group memberships the peer must satisfy.
- `identity` (String) Peer IKE identity (the 'id' value expected from the peer).
- `round` (Number) Authentication round (0 = first/default).
