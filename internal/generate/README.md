# Resource code generation

This generator (`go run ./internal/generate`, also wired via `go generate ./...`)
produces the repetitive Terraform resource boilerplate — `*_model.gen.go`,
`*_schema.gen.go`, `*_resource.gen.go` — from declarative YAML schemas under
`schemas/`. It eliminates hand-writing the four-file pattern for the many
near-identical OPNsense resources.

Generated files carry a `// Code generated by internal/generate; DO NOT EDIT.`
header. Edit the YAML schema and re-run, never the `.gen.go` files. After
generating, register the new `new<GoType>Resource` constructors in the target
package's `exports.go`.

## Schema format

```yaml
package: quagga              # target package under internal/service/
resources:
  - name: ospf_network       # file prefix + unexported identifier base (snake_case)
    go_type: OSPFNetwork     # exported Go type base -> OSPFNetworkResourceModel, newOSPFNetworkResource
    type_name: quagga_ospf_network  # Terraform type -> opnsense_quagga_ospf_network
    title: an OSPF network   # used in docs/diagnostics ("Manages <title> on OPNsense.")
    kind: item               # item (UUID CRUD) | singleton (get/set, no UUID)
    id: ospf                 # singleton only: synthetic state id
    reconfigure: /api/quagga/service/reconfigure
    monad: network           # JSON wrapper key
    endpoints:               # item: add/get/set/del/search; singleton: get/set
      add: /api/quagga/ospfsettings/add_network
      get: /api/quagga/ospfsettings/get_network
      set: /api/quagga/ospfsettings/set_network
      del: /api/quagga/ospfsettings/del_network
      search: /api/quagga/ospfsettings/search_network
    fields:
      - {name: IPAddress, tf: ip_address, json: ipaddr, type: string, required: true, desc: "..."}
```

### Field types

| type            | Go state type | OPNsense response | conversion |
|-----------------|---------------|-------------------|------------|
| `bool`          | types.Bool    | string `"0"/"1"`  | BoolToString / StringToBool |
| `int`           | types.Int64   | string            | Int64ToString / intOrZero (optional uses intOrEmpty) |
| `string`        | types.String  | string            | direct |
| `selectmap`     | types.String  | SelectedMap       | string(x); supports `options` for OneOf validation |
| `selectmaplist` | types.Set     | SelectedMapList   | sliceToSet / setToCSV |
| `csvset`        | types.Set     | string (CSV)      | CSVToSlice→sliceToSet / setToCSV |

`required: true` omits the default and marks the attribute Required; otherwise
`default` supplies the schema default. The target package must provide the
shared helpers `setToCSV`, `sliceToSet`, `intOrEmpty`, `intOrZero` (present in
`internal/service/quagga`).

Acceptance tests are not generated — add `*_resource_test.go` by hand or via
the test-arch tooling, and validate field/monad/endpoint accuracy against a
live OPNsense box.
